# Executive Briefing Generation Skill

Multi-dimensional status reporting from structured memory systems. Queries agent deliverables, active tasks, blockers, and system health to generate executive decision-maker briefings.

## When to Use

Trigger this skill when:
- Generating daily/weekly executive briefings from agent organization data
- Compiling multi-team status reports from a knowledge base (MemPalace, etc.)
- Creating decision-maker summaries aggregating deliverables, blockers, and metrics
- Running scheduled cron jobs that need to query memory and report status
- Summarizing agent operations across multiple teams or projects
- Building automated status reports that save to structured taxonomy

## Prerequisites

- Access to structured memory system (MemPalace, knowledge graph, database)
- Taxonomy understanding (wing/hall/room/closet structure)
- Delivery channel configured (Discord, email, Slack, etc.)

## Procedure

### 1. Define Briefing Scope

Identify what dimensions to query:
- **Agent Deliverables**: Yesterday's completed work by team/agent
- **Active Task Status**: Current work in progress, utilization
- **Escalations/Blockers**: Critical issues, action items
- **System Health**: Infrastructure, services, alerts
- **Strategic Metrics**: KPIs, opportunity tracking, risks

### 2. Query Structured Memory

Use semantic search + recall to gather intelligence:

```yaml
# Agent deliverables
query: "agent deliverable complete yesterday [DATE]"
wing: agent_org
hall: operations
limit: 20

# Active task status  
query: "active task status agent_org operations"
wing: agent_org
recall: room=reports

# Escalations/blockers
query: "escalation blocker critical issue failure"
limit: 10

# System health
query: "system health monitoring alert status devops_sre"
wing: agent_org
hall: operations
room: reports
closet: health_dashboard
```

### 4. Save to Structured Storage

File the report with appropriate taxonomy:

```yaml
palace: agent_org              # Target memory palace
wing: agent_org                # Organizational wing
hall: operations              # Functional domain
room: reports                  # Report category
closet: exec_briefing         # Specific report type
importance: 4                  # Executive-level: 4-5
topic: daily-executive-briefing-[YYYY-MM-DD]
tags: [exec-briefing, daily-report, agent-status, decision-support]
```

### 5. Deliver Summary

Execute delivery via configured channel:

```yaml
target: discord:#channel       # or email, slack, etc.
format: concise                # Full detail in MemPalace, summary in channel
highlight_critical: true       # 🔴 alerts first
```

## Severity Indicators

Use consistent emoji coding:
- 🔴 **HIGH/Critical**: Immediate attention required
- 🟡 **MEDIUM/Warning**: Attention soon
- 🟢 **LOW/Info**: Monitoring only
- ✅ **Resolved/Good**: No action needed
- ⏸️ **Paused/Awaiting**: Blocked or waiting
- 🔄 **In Progress**: Active work

## Escalation Patterns

| Pattern | Query | Action |
|---------|-------|--------|
| Agent stuck | "agent_name status stuck" | Flag for manager intervention |
| Resource critical | "disk usage critical" or "memory full" | 🔴 Immediate escalation |
| Deliverable missed | "overdue task" | Highlight in blockers section |
| System degraded | "service down" or "health failing" | ⚠️ Infrastructure alert |
